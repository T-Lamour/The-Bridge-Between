## Overview

n8n is used as the **automation and orchestration layer** within this SOC. It functions as a lightweight **SOAR (Security Orchestration, Automation, and Response)** platform, enabling automated handling of security alerts generated by Wazuh.

Its primary role is to reduce manual investigation effort by enriching alerts, making decisions, and triggering response actions.

---

## Key Responsibilities

* Ingest alerts from Wazuh via webhook
* Enrich alerts with threat intelligence
* Apply decision logic to determine severity and response
* Automate incident creation in DFIR IRIS
* Trigger response actions (e.g. blocking IPs, disabling accounts)

---

## Architecture Role

n8n sits between detection and response:

```id="n8n-flow"
     Wazuh SIEM
 (Alert Generation)
        │
        ▼
        n8n
 (Enrichment & Decision Engine)
        │
        ├──────────────► DFIR IRIS (Incident Creation)
        │
        ├──────────────► pfSense (IP Blocking)
        │
        └──────────────► Microsoft Entra (User Response)
```

---

## Workflow Design

Workflows in n8n are designed to follow a structured pipeline:

### 1. Alert Ingestion

* Alerts are received from Wazuh via webhook
* JSON payload includes event data such as:

  * Source IP
  * Username
  * Rule triggered
  * Timestamp

---

### 2. Data Extraction & Normalisation

* Relevant fields are extracted from the alert
* Data is formatted for consistent processing across workflows

---

### 3. Threat Intelligence Enrichment

n8n integrates with external services to provide context:

* **VirusTotal** → IP / URL reputation
* **AbuseIPDB** → Malicious IP scoring
* **MISP** → Internal threat intelligence correlation

This step determines whether an indicator is:

* Known malicious
* Suspicious
* Benign

---

### 4. Decision Logic

Conditional logic is applied to determine the next action.

Examples:

* If IP is malicious → escalate severity
* If login is successful from malicious IP → trigger response
* If false positive indicators → close or downgrade alert

---

### 5. Incident Creation

* Alerts are forwarded to **DFIR IRIS**
* A case is created with enriched data
* Relevant context is included for analysts

---

### 6. Automated Response

Depending on severity, n8n can trigger actions such as:

* Blocking IP addresses via pfSense
* Disabling user accounts in Microsoft Entra
* Revoking active user sessions
* Updating MISP with new indicators

---

## Example Use Case

### Malicious Login Detection & Response

**Scenario:**
A user successfully logs in from an IP flagged as malicious.

**Workflow Actions:**

1. Wazuh generates alert
2. n8n enriches IP using threat intelligence
3. IP confirmed malicious
4. n8n:

   * Disables user account (Microsoft Entra)
   * Revokes active sessions
   * Blocks IP via pfSense
   * Creates incident in DFIR IRIS

This reduces response time from minutes to seconds.

## Key Benefits

* Reduces manual SOC workload
* Enables consistent and repeatable responses
* Integrates multiple tools into a unified pipeline
* Highly flexible and customisable workflows
* Open-source and cost-effective

---

## Limitations

* Requires careful workflow design to avoid errors
* Debugging complex workflows can be challenging
* No native security focus (adapted for SOAR use)

---

## Design Approach

Workflows are built with the following principles:

* **Automation-first mindset**
* **Fail-safe logic** (avoid accidental disruption)
* **Modular workflows** (reusable components)
* **Scalable integrations** (easy to extend with new tools)

---

## Summary

n8n acts as the **central automation engine** of the SOC, transforming static alerts into actionable and automated responses.

It bridges the gap between:

* Detection (Wazuh)
* Intelligence (MISP, VirusTotal, AbuseIPDB)
* Response (pfSense, Microsoft Entra, DFIR IRIS)

This enables a fully automated, efficient, and scalable security operations workflow.
